Gitflow

A system we can utilize to establish process in our work, and develop in more efficient and organized ways.

Gitflow: General Rules of Thumb

  • Never branch off of master (except when creating a hotfix)
  • Always create pull requests for feature branches
  • Only releases and hotfixes should be merged into master
  • Once a release branch is made, develop is used for the next release

Semver

Major.Minor.Patch

  1. MAJOR version number when you make incompatible API changes
  2. MINOR version number when you add functionality in a backwards-compatible manner
  3. PATCH version number when you make backwards compatible bug fixes
